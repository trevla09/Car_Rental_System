# Car_Rental_System

## Use Case
Система оформлення оренди автомобіля. Користувач обирає автомобіль за ID,
система перевіряє його доступність та оформлює оренду, змінюючи статус авто.

## Обрана архітектура
**Hexagonal Architecture (Ports and Adapters)**

## Чому саме ця архітектура?
Гексагональна архітектура ідеально підходить для цього завдання, оскільки:
- Бізнес-логіка (перевірка доступності авто) повністю ізольована від технічних деталей
- Легко замінити InMemoryRepository на базу даних без зміни бізнес-логіки
- Чіткі межі між доменом, сценарієм використання та інфраструктурою

## Структура проєкту
src/main/java/kisit/demo/
├── domain/
│   └── Car.java                    — доменна сутність
├── application/
│   ├── ports/
│   │   └── CarRepository.java      — порт (інтерфейс)
│   └── usecases/
│       └── RentCarService.java     — сценарій використання
├── infrastructure/
│   └── InMemoryCarRepository.java  — адаптер (реалізація порту)
└── DemoApplication.java            — точка входу + CommandLineRunner

## Розподіл відповідальностей

| Компонент | Роль |
|---|---|
| `Car` | Domain model — сутність з бізнес-методом `canBeRented()` |
| `CarRepository` | Port — інтерфейс для взаємодії з сховищем |
| `RentCarService` | Use Case — координує бізнес-логіку оренди |
| `InMemoryCarRepository` | Adapter — технічна реалізація сховища (HashMap) |

## Бізнес-правило
Автомобіль можна орендувати лише якщо `car.canBeRented() == true`.
Якщо авто вже зайняте — викидається `IllegalStateException`.

## Як запустити

### Вимоги
- Java 21+
- Gradle (wrapper включено)

### Запуск
```bash
./gradlew bootRun

=== Тест: Оформлення оренди автомобіля ===

--- Тест 1: Оренда вільного авто (ID: 1) ---
✅ Успішна оренда: Toyota Camry (ID: 1)

--- Тест 2: Повторна оренда того ж авто (ID: 1) ---
❌ Очікувана помилка: Автомобіль 'Toyota Camry' вже зайнятий і не може бути орендований.

--- Тест 3: Оренда спочатку зайнятого авто (ID: 2) ---
❌ Очікувана помилка: Автомобіль 'BMW X5' вже зайнятий і не може бути орендований.

=== Тести завершено ===
```
